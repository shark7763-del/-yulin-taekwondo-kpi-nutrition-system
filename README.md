# 🥋 育林國中跆拳道隊｜選手 KPI＋身體狀態＋AI 飲食建議系統

> 昨天發現問題，今天帶著目標訓練，明天看見進步。

一套**純前端**（HTML + CSS + JavaScript）的跆拳道選手每日紀錄系統，可直接部署到 **GitHub Pages**，並透過 **Google Apps Script Web App** 把資料寫入 **Google Sheet**。

不需要 React / Vue / Node.js / npm / 外部資料庫 / CDN，複製即用。

---

## ✨ 核心精神

- 訓練不是只看分數，也要看身體狀態、恢復狀況與飲食習慣。
- 選手不是每天被打分數，而是每天看見自己正在成長。
- 教練不是只看誰分數低，而是知道誰需要幫助、誰需要鼓勵、誰需要晤談、誰需要注意飲食與恢復。

---

## 📦 功能總覽

- **六大分頁**：學生填寫、上次表現、個人檔案、家長後台、教練後台、系統設定
- **運動科學 KPI 30 項評分**：技術、戰術、體能、心理、態度、生理各 5 題，全部採 1–5 分正向拉桿
- **疲勞與恢復指數**：整合睡眠、睡眠品質、RPE、痠痛與身體狀態，產生恢復建議
- **上次表現回顧**：選姓名自動抓「上一筆」紀錄（不是只抓昨天）
- **今日心情指數**：學生可用 5 段表情回報狀態，教練後台彙整團隊心情與低落關懷名單
- **今日 vs 上次**：送出後比較總分、面向、體重變化
- **BMI / 體重差距**：即時計算，青少年友善、不羞辱、不鼓勵極端減重
- **規則式 AI 飲食建議**：選手版 / 家長版 / 教練版三種 + 明日小目標，並支援每餐快速勾選食物類別
- **AI 教練回饋卡**：依情境產生選手 / 家長 / 教練三版本，低心情時提醒尋求可信任成人協助
- **LINE 四版本一鍵複製**：選手 / 家長 / 教練 / 純飲食
- **未出席訓練報告**：請假原因、反思問答、補訓提醒，不再要求填 KPI
- **選手成長卡**：顯示累積紀錄、連續出席、徽章與個人座右銘
- **家長後台**：查看孩子今日狀態、最近紀錄、出席統計、補訓任務與教練提醒
- **教練後台**：今日總覽、狀態名單、團隊心情、訓練分析、飲食管理、建議晤談名單、個人最近 7 筆
- **交叉辯論**：計算透明化、自評 vs 教練評對照、選手回應與教練回覆
- **LINE 自動推播**：學生送出後自動推到教練群組（Messaging API）
- **選手名單管理**：新增 / 修改 / 刪除 / 恢復預設 / 匯入匯出（存 localStorage）
- **離線本機測試模式**：教練可不寫入 Sheet 直接測試主要流程

## 🚀 安裝與部署教學

### A. 上傳到 GitHub 並開啟 GitHub Pages

1. 登入 GitHub，點右上角 **＋ → New repository**。
2. Repository name 填 `yulin-taekwondo-kpi-nutrition-system`，設為 **Public**，按 **Create repository**。
3. 上傳檔案（兩種方式擇一）：
   - **網頁拖拉**：進入 repo → **Add file → Upload files** → 把 `index.html`、`style.css`、`app.js`、`README.md`、`sample-sheet-headers.md` 以及 `apps-script/Code.gs`（連同資料夾）拖進去 → **Commit changes**。
   - **Git 指令**：
     ```bash
     git clone https://github.com/你的帳號/yulin-taekwondo-kpi-nutrition-system.git
     # 把所有檔案放進資料夾後
     git add .
     git commit -m "init"
     git push
     ```
4. 進入 repo → **Settings → Pages**。
5. **Source** 選 `Deploy from a branch`，**Branch** 選 `main`、資料夾選 `/ (root)`，按 **Save**。
6. 等 1–2 分鐘，頁面會出現網址，例如：
   ```
   https://你的帳號.github.io/yulin-taekwondo-kpi-nutrition-system/
   ```
7. 打開網址即可看到系統；正式收資料前，仍需完成 Google Sheet / Apps Script 串接。

> ⚠️ `index.html`、`style.css`、`app.js` 必須在 repo 的**根目錄**，GitHub Pages 才找得到。

---

### B. 建立 Google Sheet

1. 到 [Google 雲端硬碟](https://drive.google.com) → **新增 → Google 試算表**。
2. 把試算表命名，例如「育林跆拳道 KPI 資料」。
3. 不需要手動貼表頭；下一步執行 `setupSheet()` 會自動建立 / 更新 `records`、`roster`、`parents`、`attendance_reports`、`appdata` 工作表。

> ⚠️ 請不要再把表頭手動貼到 A1。新版為了相容舊資料調整過欄位順序，手動貼表頭容易造成欄位錯位。

---

### C. 建立並部署 Apps Script Web App

1. 在 Google Sheet 上方選單 **擴充功能 → Apps Script**。
2. 刪掉預設的 `function myFunction(){}`，把 `apps-script/Code.gs` 全部內容貼上 → 按 💾 儲存。
3. （建議）在編輯器上方函式下拉選 **`setupSheet`** → 按 **執行**。
   - 第一次會跳授權，依序按 **檢閱權限 → 選擇你的 Google 帳號 → 進階 → 前往專案（不安全）→ 允許**。
   - 執行成功後 `records`、`roster`、`parents`、`attendance_reports`、`appdata` 工作表會自動建立 / 更新。
4. 右上角 **部署 → 新增部署**。
5. 齒輪 ⚙️ → 類型選 **網頁應用程式（Web app）**。
6. 設定：
   - **說明**：隨意，例如 `v1`
   - **執行身分（Execute as）**：**我（Me）**
   - **誰可以存取（Who has access）**：**任何人（Anyone）**
7. 按 **部署**，再次授權，完成後會得到一個 **網頁應用程式 URL**：
   ```
   https://script.google.com/macros/s/AKfyc.../exec
   ```
8. **複製這個以 `/exec` 結尾的 URL。**

> 之後若修改 `Code.gs`，要 **部署 → 管理部署 → 編輯（鉛筆）→ 版本選「新版本」→ 部署**，URL 不變。

---

### D. 把 URL 貼回系統

1. 打開你的 GitHub Pages 網址。
2. 進入 **⚙️ 系統設定** 分頁。
3. 把剛剛的 Web App URL 貼進「Google Apps Script Web App URL」欄。
4. 按 **💾 儲存設定**，再按 **🔌 測試連線**。
5. 出現「**連線成功，可以開始使用。**」就完成串接了 🎉
   - 若顯示「連線失敗」，請看下方[常見錯誤排除](#-常見錯誤排除)。

### E. LINE 自動推播（選用）

讓學生送出紀錄後，系統自動把訊息推到**教練的 LINE 群組**。用的是官方 **LINE Messaging API**（免費方案每月有一定推播額度），由 `Code.gs` 後端呼叫，**不需要額外伺服器**。

> 註：舊的 LINE Notify 已於 2025 年停止服務，因此本系統改用 Messaging API。

#### 步驟 1：建立 LINE 官方帳號與 Messaging API 頻道

1. 到 [LINE Developers Console](https://developers.line.biz/console/) 用 LINE 帳號登入。
2. 建立一個 **Provider**（隨意命名，例如「育林跆拳道」）。
3. 在該 Provider 下 **Create a new channel → Messaging API**，填基本資料建立。
4. 進入頻道 → **Messaging API** 分頁：
   - 找到 **Channel access token (long-lived)**，按 **Issue** 產生並複製（這就是 `token`）。
   - 把 **Auto-reply messages / Greeting** 視需要關閉（非必要）。

#### 步驟 2：取得要推播的「群組 ID」

群組 ID 無法用看的，要靠機器人捕獲。本系統的 Web App URL 已內建 Webhook：

1. 在頻道 **Messaging API** 分頁，找到 **Webhook URL**，貼上你的 **Apps Script Web App URL**（就是 `/exec` 那個），並把 **Use webhook** 打開。
2. 用手機掃該頻道的 **QR code**，把這個官方帳號**加進你的教練 LINE 群組**。
3. 在群組裡**隨便發一句話**（例如「test」）。
4. 回到系統 **⚙️ 系統設定 → 📱 LINE 自動推播設定**，按 **📡 自動帶入捕獲 ID**，群組 ID 就會自動填入。
   - （或在 Apps Script 編輯器執行 `getLineLastSourceId()`，從 Logger 看 ID。）

#### 步驟 3：填入設定並啟用

在 **📱 LINE 自動推播設定** 區：

1. 貼上 **Channel access token**。
2. 確認 **推播目標 ID**（步驟 2 帶入的群組 ID）。
3. 選 **推播版本**（建議「教練版」，或「教練版＋家長版」）。
4. 勾選 **啟用自動推播**。
5. 按 **💾 儲存推播設定**，再按 **🔔 測試推播一則** → LINE 群組應收到測試訊息。

完成後，學生每次「正式送出」就會自動推播到群組 🎉

#### （進階）更安全的設定方式

token 屬於敏感資訊。若不想經過公開網址，可改在 **Apps Script 編輯器** 設定：

1. 打開 `Code.gs`，找到 `setLineConfig()`，把裡面三個值改成你的 token、群組 ID、版本。
2. 函式下拉選 `setLineConfig` → 執行（授權一次）。
3. 執行 `lineTestFromEditor()` 測試。

#### （建議）加管理密碼，避免別人亂改設定

因為 Web App 是「任何人可存取」，任何知道網址的人理論上可呼叫設定。若要保護：

- 在編輯器執行一次：`PropertiesService.getScriptProperties().setProperty('ADMIN_KEY','你的密碼')`
- 之後從前端儲存／測試推播時，要在「管理密碼」欄填入相同密碼才會生效。

#### LINE 推播常見問題

| 狀況 | 解法 |
| --- | --- |
| 測試推播失敗 401 | token 錯或過期，重新 Issue 一個 long-lived token。 |
| 測試推播失敗 400 | 目標 ID 格式錯（群組是 `C...`、個人是 `U...`）。 |
| 抓不到群組 ID | Webhook URL 沒填或沒開 Use webhook；或機器人還沒被加進群組／群組沒人發言。 |
| 學生送出群組沒收到 | 「啟用自動推播」沒勾；或當月免費推播額度用完。 |
| 改了 Code.gs 沒生效 | Apps Script 要重新部署新版本。 |

## 🧑‍🎓 使用教學

### 學生填寫

1. 選手登入後會鎖定自己的姓名，系統自動帶出 **📌 上次表現回顧**、今日任務與選手成長卡。
2. 填基本資料、心情指數、身體數據（BMI 自動算）、飲食、KPI 拉桿、心得與目標。
3. 送出：
   - **🚀 正式送出到 Google Sheet**：需先設定好 URL。
   - **💬 送出並分享到 LINE**：正式送出成功後開啟 LINE 分享。
   - **💾 本機測試送出**：只在教練身分顯示，用於不寫入 Sheet 的測試。
4. 送出後會看到 **📈 今日 vs 上次**、**🍱 今日飲食回饋**、**AI 教練回饋卡**、**📱 LINE 四版本**（可一鍵複製）。

### 查看上次表現

- **📌 上次表現** 分頁選姓名 → 查最近一筆（會跳過請假日，抓真正的「上一筆」）。

### 教練後台

- **📊 教練後台** 可選日期與燈號篩選，按 **🔄 重新整理資料**。
- 看今日總覽、紅黃綠燈名單、團隊心情、全隊共同弱項、飲食風險／水量不足／宵夜名單、**建議晤談名單**。
- 下方可選個別選手查**最近 7 筆**紀錄與面向趨勢。

### 選手名單管理

- **⚙️ 系統設定 → 👥 選手名單管理**：新增、修改、刪除、**恢復預設名單**、**匯出 / 匯入 JSON**。
- 名單會先存在 **localStorage**，按 **☁️ 同步名單到雲端** 後會寫入 `roster` 工作表，其他裝置重新開啟即可取得最新名單。

### 本機測試模式

- 沒有設定 URL 時：不能正式送出；教練可用「本機測試送出」把資料存在該瀏覽器 localStorage，教練後台與上次表現會讀本機資料。

## 💬 交叉辯論功能（自評 vs 教練評）

這套系統不只是「教練單方面打分」，而是讓**教練和選手對著同一份數據一起討論**。包含三個功能：

### ① 計算透明化 🔎
- 在「📌 上次表現」和「📊 教練後台 → 個人查詢」的每一筆紀錄，都有一個可展開的 **「🔎 為什麼是這個燈號？」**
- 點開會說明：平均怎麼算、哪些面向／細項把分數拉低、紅黃綠燈的門檻
- 用意：選手看得懂規則，才有能力跟教練討論

### ② 自評 vs 教練評對照 ⚖️
**教練操作：**
1. 進 **📊 教練後台 → 👤 個人查詢**，選選手 → 按「查詢」
2. 下方「✍️ 評分與對話」會列出該選手最近 7 筆
3. 對某一筆點開 **「✍️ 填寫教練評分」**，拉六大面向的分數（旁邊會顯示選手自評多少）、寫評語
4. 按 **💾 儲存教練評分**

**結果：**
- 系統自動並排顯示「選手自評 vs 教練評」與差距
- 差距 ≥ 1 分的面向會標 **💬「值得討論」**
- 選手在「上次表現」也看得到這個對照

### ③ 選手回應與教練回覆 💬
**選手操作：**
- 在「📌 上次表現」查自己 → 最下方 **「💬 我對這筆的看法」** 寫下想法 → 送出
- 例如：「核心穩定我覺得不只 2 分，因為今天…」

**教練操作：**
- 教練後台「個人查詢」會看到選手的看法 → 在 **「💬 回覆選手」** 回應 → 送出
- 選手回到「上次表現」刷新就看到教練回覆

### 一條龍測試流程
1. 學生填一筆新紀錄並「正式送出」（要有 `recordId`，舊紀錄不支援）
2. 教練後台 → 個人查詢 → 對那筆評分＋寫評語＋儲存
3. 學生 → 上次表現 → 看到對照、寫下看法
4. 教練 → 看到看法 → 回覆
5. 學生 → 刷新上次表現 → 看到回覆

### ⚠️ 設計原則
- **透明只對「自己的資料」**：選手看得到教練怎麼評他、為什麼，但**看不到別人的分數**（避免互相比較）
- **不是討價還價**：回應欄是讓選手「說明想法、幫助教練了解」，**不是用來改分數**，最終由教練綜合判斷
- 舊紀錄（沒有 recordId）無法評分／回應，屬正常現象

> 📌 **重要**：使用這些功能前，後端 `Code.gs` 要**重新部署新版本**，並執行一次 `setupSheet()` 讓 Google Sheet 自動補到最新欄位。

## 🍱 AI 飲食建議說明（規則式，非醫療）

系統用 `analyzeNutrition()` 依關鍵字判斷：蛋白質、蔬菜、水量、含糖飲料、油炸、主食、宵夜、訓練強度，產生三種正向建議。

**安全原則**：不做醫療診斷、不鼓勵極端節食、不叫學生不吃飯、不用羞辱字眼、不把體重當唯一標準；青少年提醒不要過度減重；體重變化過大時提醒教練與家長關心。

## 🔐 資料安全提醒（請務必閱讀）

1. **GitHub Pages 是公開網頁**，任何人知道網址都能打開。
2. **不要在程式碼或頁面放敏感個資**（身分證、電話、住址等）。
3. **Web App URL 若公開**，任何知道網址的人都可能送資料或讀資料。若需保護，可日後在 `Code.gs` 與前端加上簡單密碼 / token 驗證。
4. **身高體重屬於較敏感資料**，請教練妥善管理，不要公開分享個人資料。
5. **家長版回覆**請避免公開貼在群組互相比較孩子體重。
6. 建議把 Google Sheet 的分享權限維持在「僅限你或指定教練」。

## 🛠 常見錯誤排除

| 狀況 | 可能原因與解法 |
| --- | --- |
| 測試連線「連線失敗」 | URL 沒以 `/exec` 結尾；部署存取權不是「任何人」；用到舊版部署。重新「管理部署 → 新版本」。 |
| 送出沒進 Sheet | Web App URL、部署權限或新版部署有問題；先執行 `setupSheet()`，再重新部署 Apps Script 新版本。 |
| 上次表現抓不到 | 該選手還沒有任何紀錄（正常，會顯示「第一筆紀錄」提示）。 |
| 點「正式送出」叫我去設定 | 尚未在系統設定貼 URL，或 `CONFIG.WEB_APP_URL` 留空；先完成步驟 D。 |
| 改了 Code.gs 沒生效 | Apps Script 要重新部署新版本。 |
| 名單改了學生選單沒更新 | 重新整理頁面；名單存在該瀏覽器 localStorage，換裝置不會同步。 |
| 中文變亂碼 | 確認檔案以 UTF-8 儲存（本專案皆為 UTF-8）。 |

## 🧮 分數規則

- 新紀錄固定 30 項 KPI，每細項 1–5 分；總分 = 30 項加總（滿分 150）；六面向各取 5 題平均。
- 面向平均依各 KPI 歸屬彙總，用於回顧、趨勢與紅燈原因分析；舊資料仍保留滿分 30 顯示。
- 狀態：平均 ≥ 4.0 🟢 綠燈；3.0 ≤ 平均 < 4.0 🟡 黃燈；平均 < 3.0 🔴 紅燈。
- 低分細項：所有 < 3 分的細項，取最低三項（同分依出現順序）。

## 📡 API 格式（前端 POST JSON 給 Web App）

新增紀錄：
```json
{ "action": "addRecord", "payload": { "...": "..." } }
```
查上一筆：
```json
{ "action": "getLastRecordByName", "name": "林子棠" }
```
查最近 7 筆：
```json
{ "action": "getRecentRecordsByName", "name": "林子棠", "limit": 7 }
```
查今日紀錄：
```json
{ "action": "getTodayRecords", "date": "2026-06-02" }
```
測試連線：
```json
{ "action": "ping" }
```
回傳統一格式：`{ "ok": true, "data": ... }` 或 `{ "ok": false, "error": "..." }`。
